**日期**: 2026-05-02
**状态**: 草稿

---

## 核心原则

> **铁律**: 安全不是后置审查，而是前置的、不可逾越的规则。

### 违反规则的代价

| 违反行为 | 处理方式 |
|----------|----------|
| 敏感数据进入日志 | 立即修复，审查日志输出 |
| 硬编码密钥 | 替换为环境变量，拒绝合并 |
| 权限校验遗漏 | 拒绝合并，退回补全 |

---

## 敏感数据处理红线

**绝对禁止** 以下数据进入任何日志输出（`logger`、`print`、APM 监控）：

| 敏感字段 | 示例 |
|----------|------|
| 密码 / Password | 用户登录密码 |
| Token / 令牌 | API 密钥、会话令牌 |
| 手机号 | 患者联系方式 |
| 身份证号 | 证件号码 |

### 错误示例

```python
# 违反安全红线
logger.info(f"API key: {api_key}")
logger.debug(f"Token: {token}")
logger.info(f"Phone: {phone}")
```

### 正确做法

```python
# 使用脱敏日志
logger.info(f"API key: ***")
logger.debug(f"Token: ***")
```

## 权限校验点

权限校验统一在 API 网关层 / 中间件层处理，**不得**在业务逻辑中遗漏：

| 层级 | 职责 |
|------|------|
| API 网关 | 认证令牌验证、请求速率限制 |
| 中间件 | 请求上下文注入、权限标志传递 |
| 业务逻辑 | 基于已注入的权限标志做数据过滤 |

---

## 安全开发原则

### 零信任输入

不信任任何外部数据：
- API 响应
- 用户输入
- 文件内容
- 环境变量

### 验证在边界

所有用户输入在系统边界处立即验证，快速失败并给出清晰的错误消息。

### 最小权限

每个模块只申请所需的最小权限，不过度授权。

---

## 安全检查清单

完成实现前，确保每项都已检查：

- [ ] 无硬编码密钥（API 密钥、密码、令牌）
- [ ] 所有用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] 错误消息不泄露敏感信息
- [ ] 敏感数据未进入日志
