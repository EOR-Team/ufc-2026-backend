# CODING 编码规范

**日期**: 2026-05-02
**状态**: 草稿
**适用**: Vibe Coding 模式下的 AI 自动开发

---

## 核心原则

> **铁律**: 无类型注解的函数禁止提交
>
> "If it's not typed, it's not done."

### 强制规则

**AI 在以下场景必须询问人类，不得自行决定**：
- 需要突破目录结构组织方式时
- 需要添加新的顶层模块时
- 涉及外部服务集成方式时

### 违反规则的代价

| 违反行为 | 处理方式 |
|----------|----------|
| 提交无类型注解的函数 | 拒绝合并，退回重写 |
| 组织结构违反 feature/domain 原则 | 拒绝合并，重构目录 |
| 使用 print() 代替 logger | 强制替换，逐处检查 |
| 硬编码配置值 | 强制提取为常量或配置 |

---

## 代码组织

### 目录结构原则

**按 feature/domain 组织，禁止按类型组织**：

```
# Good - 按功能组织
src/
├── medical_care.py      # 医疗建议模块
├── triager/
│   ├── clinic_selector.py
│   ├── condition_collector.py
│   └── route_patcher.py

# Bad - 按类型组织
src/
├── agents/              # 禁止按类型划分
├── utils/
└── ...
```

### 文件大小限制

- **单个文件**: 最多 800 行
- **单个函数**: 最多 50 行
- **超出时**: 必须拆分为更小的模块/函数

---

## Python 标准

### PEP 8 + 项目扩展

| 规则 | 标准 |
|------|------|
| 行长度 | 最多 120 字符 |
| 缩进 | 4 空格（禁止 Tab） |
| 顶级定义间空行 | 2 行 |
| 函数定义间空行 | 1 行 |

### 类型注解（强制）

**所有函数签名必须有类型注解**：

```python
# Good
def get_medical_care_advice(symptoms: str, diagnosis: str) -> tuple[dict, object]:
    ...

# Bad - 无类型注解
def get_medical_care_advice(symptoms, diagnosis):
    ...
```

**类型注解检查**：
- 参数必须标注类型
- 返回值必须标注类型
- 使用 `mypy` 或 IDE 检查

---

## 注释规范

### 注释要求

**使用中文注释，以 `# ` 格式**：

```python
# 这是中文注释
result = load_model(model_id, filename)  # 加载模型

# Bad - 英文注释
# This is a comment
result = load_model(model_id, filename)
```

### 注释位置

```python
# Section header
# ========================================
# 模型管理
# ========================================

# Important warning
# 注意: 不要在此处修改
```

## AI 行为边界

### 强制询问条款

**AI 在 Vibe Coding 模式下必须询问人类的场景**：
1. 需要突破目录结构时（新建顶层目录）
2. 需要添加新的外部依赖时
3. 遇到模糊的类型注解需求时
4. 违反铁律但不确定如何处理时

### 边界规则

1. **类型优先** - 所有函数必须有类型注解
2. **结构固化** - 按 feature 组织目录结构
3. **日志规范** - 禁止 print()，使用 logger
4. **常量提取** - 配置值必须提取为常量
5. **大小限制** - 函数 50 行、文件 800 行为硬限制

---

## 常见错误与处理

| 错误 | 处理方式 |
|------|----------|
| 无类型注解 | 立即添加，拒绝合并 |
| 按类型组织目录 | 重构为按 feature 组织 |
| 使用 print() | 替换为 logger.info/warning/error |
| 硬编码值 | 提取为模块级常量 |
| 函数过长 | 拆分为更小的函数 |
| 文件过大 | 按功能拆分为多个模块 |
