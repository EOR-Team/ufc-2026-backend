# Changelog 规范

## 目的

记录项目中的重要决策，避免重复发明轮子。每个条目回答"为什么当时没那样做"。

---

## 场景一：创建新的 ADR 条目

### 规则

1. **确定序号**：使用当前最大序号 + 1，三位数字不足补零
2. **确定标题**：简洁描述决策内容，使用中文，snake_case 命名
3. **填写日期**：格式 `YYYY-MM-DD`，使用决策当天日期
4. **填写决策者**：只写人类参与者，不记录 AI
5. **背景描述**：说明约束条件和决策动机
6. **列出方案**：至少 2 个方案对比，包含评估维度（内存、质量、兼容性等）
7. **明确决策**：用 `**选择:**` 格式突出最终选择（冒号前无空格）
8. **陈述理由**：列出 2-3 个选择的主要原因
9. **说明放弃**：解释为什么放弃其他选项（如果有）
10. **后续行动**：用 `[ ]` 标记待办，不放已完成的项

### 示例模板

```markdown
# 001: 示例标题

**日期**: 2026-05-02
**状态**: 已通过
**决策者**: n1ghts4kura

## 背景

需要为 xxx 场景选择方案，要求：
- 约束 A
- 约束 B

## 考虑的方案

| 方案 | 维度1 | 维度2 | 结论 |
|------|-------|-------|------|
| A    | ...   | ...   | 候选 |
| B    | ...   | ...   | **放弃** |

## 决策

**选择:** 方案 A

## 理由

1. 原因一
2. 原因二

## 放弃的替代方案

- **方案 B**: 放弃原因

## 后续行动

- [ ] 待办事项 A
```

## 场景二：修改已有的 ADR 条目

### 规则

1. **更新日期**：将 `**日期**` 改为修改当天
2. **更新状态**：
   - 如果决策仍有效，保持 `已通过`
   - 如果决策被废弃，改为 `已废弃`
   - 如果被新决策替代，改为 `已替换` 并注明替代的序号
3. **更新决策者**：追加新的参与者，保留原有人员

### 修改还是新建？决策流程

在修改已有 ADR 前，必须先思考：**应该修改现有条目，还是创建新条目？**

#### 应该修改现有条目的情况

- **修正错误**：现有条目内容有事实性错误需要更正
- **补充遗漏**：遗漏了重要的理由、替代方案或约束条件
- **完善后续行动**：添加已完成的行动标记

#### 应该创建新条目的情况

- **推翻决策**：之前的决策现在被证明不正确，需要记录新的决策
- **新增约束**：新增了之前没有考虑的重要约束条件
- **方案升级**：发现了明显更好的替代方案

#### 判断角度

1. **时间相关性**：新信息是否改变了当初决策的前提？
2. **决策性质**：是"补充说明"还是"推翻重来"？
3. **历史价值**：保留原始决策上下文是否有助于理解项目演进？

## 文件命名

```
{序号}_{简短标题}.md
```

- 序号：3 位数字
- 简短标题：snake_case，中文描述

示例：`001_tts_solution.md`

---

## 字段说明

| 字段 | 必须 | 说明 |
|------|------|------|
| 日期 | 是 | 决策日期 |
| 状态 | 是 | `已通过` = 当前有效，`已废弃` = 不再使用，`已替换` = 被新 ADR 替代 |
| 决策者 | 是 | 参与决策的人员（不记录 AI） |
| 背景 | 是 | 约束条件和决策动机 |
| 考虑的方案 | 是 | 至少包含 2 个方案对比 |
| 决策 | 是 | 明确的选择 |
| 理由 | 是 | 选择的 2-3 个主要原因 |
| 放弃的替代方案 | 否 | 解释为什么放弃其他选项 |
| 后续行动 | 否 | 待完成的事项，用 `[ ]` 标记 |

---

## 序号冲突处理

当需要重新启用旧序号时（如重新考虑一个被废弃的决策），使用 `_v2` 后缀：

```
001_v2_tts_solution.md
```

而非覆盖或删除旧文件。每个版本都是完整的历史记录。

---

## 反模式

- 不要用 `决策者` 字段记录 AI 名称
- 不要在 `后续行动` 中放置已完成的项
- 不要删除已通过的 ADR，而是将状态改为 `已废弃`
- 不要为了"保持整洁"合并历史决策，每条 ADR 独立存在
- **禁止删除**：ADR 文档创建后，任何理由都不得删除 — changelog 是 markdown 版的 git，历史不可抹去
